### Hands-on Practice

### Implemented Queries

✔ Employee and Manager relationships using Self Join  
✔ Employee and Department mapping

✔ Employees without projects  
✔ Departments without employees  
✔ Employees reporting to managers  
✔ Employee count by department  
✔ Multiple table joins (Employee + Department + Project)

### Key Understanding

- `inner join` returns only matching records.
- `left join` returns all rows from the left table and matching rows from the right table.
- `right join` returns all rows from the right table and matching rows from the left table.
- `full outer join` combines left and right joins.
- `self join` is used when a table references itself.

### Key Understanding

- `round()` → rounds values.
- `ceil()` → rounds up.
- `floor()` → rounds down.
- `truncate()` → removes decimal values without rounding.
- `mod()` → returns remainder.
- `power()` → calculates exponent values.
- `greatest()` → returns highest value.
- `least()` → returns lowest value.
